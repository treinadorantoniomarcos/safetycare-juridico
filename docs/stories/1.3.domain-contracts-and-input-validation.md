## User Story

**As a** plataforma SAFETYCARE,  
**I want** definir contratos e validacoes para intake, consentimento e auditoria,  
**so that** web, worker e banco compartilhem fronteiras consistentes.

## Acceptance Criteria

1. Existem schemas Zod para lead de entrada, consentimento, criacao de caso e evento de auditoria.
2. Os tipos compartilhados estao centralizados em package reutilizavel.
3. As validacoes podem ser reutilizadas por web intake, webhook e worker.
4. Nenhum payload externo e aceito sem passar por schema.
